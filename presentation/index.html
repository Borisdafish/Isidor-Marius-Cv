BetterTanks

2D Tile-Based Tank Battle Game — Unity WebGL

Presented by Isidor Marius

Project Overview

BetterTanks is a 2D top-down tank battle game built in Unity. Players navigate a destructible tile-grid map, engage enemies, and complete objectives across two distinct game modes.

  • Genre: 2D Top-Down Shooter
  • Engine: Unity 6000.5.1f1 (C#)
  • Platform: WebGL (GitHub Pages)
  • Team size: Multi-developer (Git Flow)

Game Modes

Classic Mode

Classic 1v1 tank combat on a symmetrical tile-grid arena.

  • Player vs. BasicEnemy waves
  • 4-directional movement & shooting
  • Wall destruction for tactical cover
  • Score tracking & game-over screen

Script: ClassicMode.cs

Base Defense Mode

Defend your headquarters against escalating enemy waves.

  • BaseHQ takes damage on contact
  • Progressive wave difficulty
  • 3 lives represented by tank icons
  • Lose condition when base is destroyed

Script: BaseDefenseMode.cs

Architecture Overview

Component-based, data-driven architecture with centralized managers.

Core/ — GameManager, GridManager, AudioManager, ObjectPool Player/ — PlayerController, PlayerShooting, PlayerInput Enemies/ — BasicEnemy, EnemySpawner, EnemyAI Combat/ — Bullet, BulletPoolManager, ImpactAnimation Terrain/ — TileManager, DestructibleWall UI/ — MainMenu, HUD, PauseMenu, GameOverScreen Effects/ — MuzzleFlashEffect, TankExplosion Data/ — GameConfig (ScriptableObject)

Key Classes

Core Systems

  • GameManager — global state, mode switching
  • GridManager — tile ↔ world coordinate mapping
  • TileManager — wall queries, damage application
  • ObjectPool — generic reusable object pool
  • BulletPoolManager — bullet lifecycle
  • AudioManager — SFX & music playback

Actors

  • TankController — movement abstraction
  • PlayerController — input-driven tank movement
  • PlayerShooting — cooldown, bullet spawn, muzzle VFX
  • BasicEnemy — AI brain, TakeDamage()
  • EnemySpawner — wave management
  • BaseHQ — destructible objective

Combat System

Bullet.cs handles the full projectile lifecycle:

  • Direction-based rotation on spawn
  • Speed & lifetime management
  • Wall collision → DamageWall() + impact VFX
  • Tank collision → TakeDamage() + return to pool
  • Out-of-bounds → silent return

Object pooled via BulletPoolManager

Two bullet types:

  • BulletType.Scout — fast, small sprite
  • BulletType.Infantry — slower, larger sprite

Impact Animation System

Frame-by-frame sprite animations stored in Resources/Sprites/Effects/Impacts/ — Scout (5 frames) and Infantry (4 frames). Falls back to a procedural explosion sprite if no frames are found.

Tile & Destructible Environment

  • GridManager — manages the tile grid, provides WorldToTile(), IsInBounds(), IsBorderTile()
  • TileManager — per-tile queries: IsWall(), DamageWall(), wall health tracking (full → damaged → destroyed)
  • Walls have 2 visual states: wall_full.png and wall_damaged.png
  • Bullets deal 1 damage per hit; walls have configurable HP pool
  • Border tiles act as soft map boundaries — bullets bounce / despawn

UI System

Screens

Web Deployment (GitHub Pages)

  • Custom BuildWebGL.cs editor script — builds scenes: Boot, MainMenu, Gameplay
  • Output: Builds/WebGL/ with .wasm.unityweb, .data.unityweb, .framework.js.unityweb
  • Custom index.html with:
    • Dark gradient background placeholder
    • "Play Game" button → creates Unity instance
    • Loading bar with progress callback
    • Trailer video placeholder (16:9 box)
  • Hosted on GitHub Pages (branch MariusDev)
  • Live URL: crs011.github.io/proiectul-Unity-2D
